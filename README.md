## World Trade Associations

This project generates a dataset for trade associations from the book volume _World Guide to Trade Associations_, published in 2002. The book pages were scanned to generate a jpg image file for each page. These are the inputs for
the ocr_pipeline.py script. The script sends each scanned page image to the Gemini API and extracts structured association data (country, name, address, focus, ID number).

Progress is saved after every page, so the `ocr_pipleline.py` script can be safely interrupted and restarted. Pages already processed are skipped automatically.

The generated CSV file is cleaned with the script `ocr_cleanup.py`. Errors are then checked with `ocr_error_check.py`, which outputs a pages_to_redo.txt file. That file can be manually edited, and errors can be decreased by re-submitting the list of files to a superior Gemini model, using `error_redo.py`.

**Warning: do not interrupt error_redo.py.** The script removes all selected pages from `progress.jsonl` at startup, before any OCR begins. If it is interrupted, those pages are left unprocessed and absent from the checkpoint. Recovery: run `ocr_pipeline.py` without any arguments. This will detect and reprocess all pages missing from the checkpoint. (Unlike `error_redo.py`, `ocr_pipeline.py` itself is safe to interrupt because it saves progress after every page.)

If you want to process only a subset of pages listed in `pages_to_redo.txt`, edit and save the file **before** running `error_redo.py`, or use the `--top N` or `--min-issues N` flags to redo a certain number from the top of the ranked list. 

To compare models on a sample batch or sample file, the optional script `model_comparison_test.py` can be used.

The image file collection contained some duplicate scans (different image number but same content, not always sequentially numbered) and these were detected by running `find_duplicate_pages.py`. This script can work either on the entire collection or on the list of image files generated by ocr_error_check.py. Finding duplicates by cleaning the output data file is incomplete because there is too much inconsistency in OCR, so that many duplicates do not match exactly. Two pages were initially missing from the image collection. The missing data was detected by the error analysis in `ocr_error_check.py`.

Many association names were not in English. To help categorize them via both name and focus area, the names were translated to English using `translate_names.py`.

For now the repository is missing the scanned book pages (WorldGuideTrade_bookpages directory) due to the size of the image file collection.

Dependencies — install in your virtualenv:

```
pip install google-genai Pillow python-dotenv
```

Usage — run the scripts sequentially:

```
python ocr_pipeline.py
python ocr_cleanup.py
python ocr_error_check.py
python error_redo.py
python translate_names.py
```

You need an API key from Google AI Studio. Place the key in an .env file within your working directory. See the example file, .env.example. Be sure to add this file to .gitignore.
